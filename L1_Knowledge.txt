Terms

    knowledge-based agents - Agents that reason by operating on internal representations of knowledge.
        - Use known information to produce conclusions or new information.

    Sentence - An assertion about the world in a knowledge-representation language,
        such as propositional logic. A statement about the world.

    Model - Assigns a truth value (true or false) for a possible world.

    Knowledge base - A set of sentences held by a knowledge-based agent.

    Entailment - alpha |= beta:
        In every possible world where alpha is true, beta is also true.

    Inference - The process of deriving new sentences from old ones.

    Knowledge engineering - The process of deciding which symbols and logic to use and how to represent knowledge.

    Clause - A disjunction ("or") of literals (propositional symbols or their negations). (Conjunction is "and".)

    Conjunctive normal form (CNF) - A logical sentence that is a conjunction of clauses.

Logical connectives used in the computer's reasoning

    Symbols - Each symbol is either true or false.

        --|  not  (negation)  — the opposite
                Example: --| raining  means "not raining"

        ^  and  (conjunction)
                Example: false ^ true returns false

        v  or  (disjunction)
                Example: false v true returns true
                Only returns false when both are false

        -->  implication
                If P --> Q, the implication is false only when P is true and Q is false

        <-->  biconditional (iff)
                True when P and Q have the same truth value

Checking entailment / models

    Model checking - To test entailment by enumerating all possible models (less efficient).
                    If in every model where the KB is true the query is also true, then KB entails the query.

Inference rules

    Modus Ponens -
        If A --> B and A is true, then infer B.

    And elimination -
        From A ^ B, infer A (or infer B).

    Double negation elimination -
        --|(--|A) is equivalent to A.

    Implication elimination -
        A --> B is equivalent to --|A v B.

    Biconditional elimination -
        A <--> B is equivalent to (A --> B) ^ (B --> A).

    De Morgan's laws -
        --|(A ^ B) is equivalent to (--|A) v (--|B)
        --|(A v B) is equivalent to (--|A) ^ (--|B)

    Distributive property -
        Use to transform between nested conjunctions and disjunctions when converting to CNF.

    Conflict (complementary literals) -
        If one clause contains P and another contains --|P, resolving them forces the other disjuncts to be true (or leads to contradiction).

Theorem proving

    Initial state: starting knowledge base
    Actions: inference rule applications
    Transition model: new knowledge base after inference
    Goal test: check whether the target statement has been proved
    Path cost function: number of steps in the proof

Conversion to CNF (Conjunctive Normal Form)

    - Eliminate biconditionals: A <--> B  ->  (A --> B) ^ (B --> A)
    - Eliminate implications: A --> B  ->  --|A v B
    - Move negation inward using De Morgan's laws (push nots to literals)
    - Distribute v over ^ using distributive law to obtain conjunction of disjunctions
        Example:
            (P v Q) --> S
            becomes --|(P v Q) v S
            which is (--|P ^ --|Q) v S
            which distributes to (--|P v S) ^ (--|Q v S)
    - Remove duplicate literals in a clause: (Q v W v E v E) -> (Q v W v E)
    - If you derive both P and --|P as unit clauses, that yields the empty clause (contradiction).

Resolution (proof by contradiction)

    To determine if KB entails A:
    - Convert KB ^ --|A to CNF.
    - Repeatedly apply resolution to derive new clauses.
    - If you derive the empty clause, the set is unsatisfiable, so KB entails A.
    - If no empty clause can be derived, entailment is not proven.

First-Order Logic (for richer knowledge representation)

    - Contains constants and predicates; predicates applied to constants produce truth values.
        Example: predicate(constant)

    Quantifiers:
        Universal quantifier (∀) — true for all values.
        Existential quantifier (∃) — true for at least one value.

Notes / tips

    - Use conjunctions (^) to list multiple pieces of knowledge.
    - Use negation (--|) to express exclusion.

Logic puzzles

    - If something implies something else, use negation to exclude contradictory possibilities.
    - If an object occupies a position, others cannot occupy the same position.
