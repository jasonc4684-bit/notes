# Terms

Supervised Learning - given access to an input-output pair, learn a function to map inputs to outputs

    Classification - given input, mapping to a discrete category

        Such as a table with data,
        which can be the function's parameters, f(C1, C2) or hw(w is the weight) (C1, C2) to hypothesize

            The weight for C1 and C2 will vary, and hypothesize,
                such as Rain if C0(some other factors) + W1C1 + W2C2 >= 0, else opposite
                or                  hw(x)            w * x >= 0

            Perceptron Learning Rule/weight update - Wi = Wi + a(y - Hw(X)) + Xi ,
                                                        a is learning rate, y - Hw(X) = actual value - estimate

            Threshold function (hard) - A certain part of the graph where the output will change suddenly from the dot product
                Only possible with two outputs: 0 and 1

            Logistic function (soft) - A curved graph where the output changes by dot product, less absolute
                0 <= C <= 1

            1. K-nearest-neighbor classification (variant k added) -
                    Choose the category of the input based on the k-range most common knowledge table/info

                Downside - slow in organizing and measuring data

            2. Linear approach - a line separating two datasets, only for clean data

            3. Support Vector Machines - Utilizes a maximum margin separator, a boundary that maximizes the distance between data
                Such as a circle instead of a straight line


    Regression - given input, mapping continuous value/output

        ex. f(advertising) = 6000 profits
            h(advertising) the profits from different advertising value

        Loss function - expresses how poorly our hypothesis performs

            0-1 loss function -
                L(actual, predicted) = 0 if actual = predicted, 1 otherwise

            L1 loss function -
                L(actual, predicted) = |actual - predicted|

            L2 loss function -
                L(actual, predicted) = (actual - predicted)^2 , more harsh and penalizes



    Over-fitting - A model that fits too closely to a data set and may fail to generalize to future data

    Regularization - cost(h) = loss(h) + complexity(h) * P, parameter for weights/penalize

    Holdout cross-validation - splitting data into a training and test set

    k-fold cross-validation - splitting data into k sets and experimenting k times

    scikit-learn modules for test and train, for quicker data swap

        zip() allows inspecting two different lists
        .fit() for training
        .predict() to test
        X_1, X_2, Y_1, Y_2 = train_test_split(X_label, Y_label, test_size= )

Reinforcement Learning - Learning from past experiences, rewards | punishment, to take future actions

    Agent will be at a state given by environment
    when agent performs action, two outcomes: reward and punishment
        Agent has no information at initial state

    Markov decision process - model for decision-making, representing states, actions, and their rewards
        Set of states S 
        set of actions Actions(s)
        Transition model P(s' | s, a), probability at next state given current state and action
        Reward function R(s, a, s'), reward on this process

    Q-Learning - estimate the reward from Q(s, a)

        Q(s, a) = 0 for all s, a
        When action a taken in state s and receive reward r:
            Estimate the value Q(s, a) based on current and expected future rewards
            Update Q(s, a) with old estimate and new estimate

        Q(s,a ) <- Q(s, a) + a(new_estimate - old estimate)  a controls the importance of new information
                                r + future r      Q(s, a)      future r (reward) = y Max a' Q(s', a')  y=rate
                                                                Greedy decision-making

    function approximation - now explore all of the possible states, approximate from previous similar states

    Explore vs. Exploit
        Exploring additional routes|solutions to reach the goal, while vs. exploiting known route and continue

        Potential solutions
            e-greedy - set e to random move frequency
            with probability 1 - e , choose estimated best move
            with probability e, choose random move

Unsupervised Learning - given input data without any feedback or label, but still would learn patterns 

    Clustering 
        organizing objects into groups in a way that similar objects tend to be in 

        k-means clustering
            First with k clusters, repeatedly updating data points' centers and points, forming groups
                clusters move to the center of its grouped points
                if some points were far from its original cluster and closer to others, 
                    those points will become the nearest cluster's points
